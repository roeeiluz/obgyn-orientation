# מדריך פריסה — אפליקציית אורינטציה OB/GYN

מדריך זה מסביר שלושה מסלולים, מהפשוט לחי-ברשת. כולם מתבססים על קובץ אחד: `אורינטציה_אפליקציה.html`.

מה שכבר עובד עכשיו: פתיחת הקובץ בדפדפן נותנת אפליקציה מלאה ששומרת הכל מקומית בדפדפן. כל הצעדים למטה הם רק כדי להעלות אותה לרשת ולסנכרן בין מכשירים.

---

## מסלול A — שימוש מקומי (אפס התקנה)

לחיצה כפולה על `אורינטציה_אפליקציה.html` פותחת אותה בדפדפן. בחר/י כניסה (מתמחה או מנהל), והכל עובד: לוח, צ'קליסטים, הערות, ייצוא.

הערה: הקישורים למסמכים (פרוטוקולים ו-GABBE) עובדים כל עוד הקובץ יושב באותה תיקייה שבה נמצאות התיקיות `פרוטוקולים מיילדות` ו-`textbook gabbe` — בדיוק כמו עכשיו.

מגבלה: הנתונים נשמרים רק בדפדפן הזה. אם מתמחה פותח בטלפון שלו — הוא מתחיל מנתונים נפרדים. כדי שהערות המתמחה יופיעו אצלך — המשך למסלול C.

---

## מסלול B — העלאה לרשת דרך GitHub (כתובת אינטרנט, עדיין פר-מכשיר)

1. פתח/י חשבון חינמי ב-[github.com](https://github.com) וב-[netlify.com](https://netlify.com).
2. ב-GitHub: New repository → שם (למשל `obgyn-orientation`) → Private → Create.
3. גרור/י לחלון ההעלאה של GitHub את הקובץ `אורינטציה_אפליקציה.html` ואת התיקיות `פרוטוקולים מיילדות` ו-`textbook gabbe` (כדי שהקישורים יעבדו גם ברשת). שנה/י את שם הקובץ הראשי ל-`index.html` כדי שייפתח אוטומטית.
4. ב-Netlify: Add new site → Import from GitHub → בחר/י את ה-repository → Deploy.
5. תוך דקה תקבל/י כתובת (למשל `https://obgyn-orientation.netlify.app`) שאפשר לשלוח לכל אחד.

עדיין: אין סנכרון בין מכשירים. כל מכשיר עם הנתונים שלו. למצב חי — מסלול C.

---

## מסלול C — מצב חי רב-מכשירים עם Supabase (מומלץ)

זה מה שהופך אותה למה שביקשת: מתמחה רושם הערה בטלפון → היא מופיעה אצלך במסך, וכולם רואים אותו לוח מעודכן.

### שלב 1 — יצירת פרויקט Supabase (חינמי)
1. היכנס/י ל-[supabase.com](https://supabase.com) → Sign in → New project.
2. תן/י שם וסיסמה למסד הנתונים → Create. המתן/י כדקה עד שהפרויקט מוכן.

### שלב 2 — הקמת הטבלה
1. בתפריט הצד: **SQL Editor** → New query.
2. פתח/י את הקובץ `supabase_schema.sql` (כאן בתיקייה), העתק/י את כל תוכנו, הדבק/י, ולחץ/י **RUN**. אמורה להופיע הודעת הצלחה.

### שלב 3 — העתקת המפתחות
1. בתפריט הצד: **Project Settings → API**.
2. העתק/י שני ערכים:
   - **Project URL** (משהו כמו `https://abcdxyz.supabase.co`)
   - **anon public** key (מחרוזת ארוכה)

### שלב 4 — הדבקת המפתחות באפליקציה
1. פתח/י את `אורינטציה_אפליקציה.html` בעורך טקסט (או בקש/י ממני).
2. בראש קטע ה-`<script>` יש בלוק:
   ```
   var CONFIG={
     SUPABASE_URL:'',
     SUPABASE_KEY:''
   };
   ```
3. הדבק/י בתוך הגרשיים את שני הערכים מהשלב הקודם. שמור/י.

זהו — מרגע זה האפליקציה טוענת ושומרת בענן, ומתרעננת חי בין מכשירים. אם תשאיר/י את הערכים ריקים, היא חוזרת אוטומטית למצב מקומי.

### שלב 5 — העלאה לרשת
חזור/חזרי על מסלול B (GitHub + Netlify) עם הקובץ המעודכן. כעת הכתובת שתשלח/י — כולם רואים בה את אותם הנתונים החיים.

---

## הערות

**קבצי המסמכים (פרוטוקולים / GABBE).** שתי אפשרויות:
- הפשוטה: להעלות את התיקיות יחד עם הקובץ ל-GitHub (כמו במסלול B). הקישורים היחסיים ימשיכו לעבוד.
- המסודרת: להעלות את ה-PDF ל-**Supabase Storage** ולעדכן במסך "תוכנית למידה עיונית" → ערוך מקור → להדביק את ה-URL הציבורי של הקובץ. כך הקבצים מנוהלים יחד עם הנתונים.

**אבטחה ופרטיות.** אין כאן מידע רפואי של מטופלים. עם זאת, מפתח ה-anon הוא ציבורי — כל מי שיש לו את הכתובת יכול לקרוא ולערוך. זה תואם למודל השקוף ומבוסס-האמון שהגדרת (1–2 מתמחים בתיאום הדוק). אם בעתיד תרצה הגבלה אמיתית (התחברות אישית, מנהל בלבד עורך) — זה שדרוג נקי שאפשר להוסיף.

**גיבוי.** Supabase מגבה אוטומטית. בנוסף, מסך "הגדרות וייצוא" באפליקציה מאפשר בכל רגע להוריד CSV, יומן (.ics) ותיק ביצוע.

## מה אני ממליץ לעשות כשתתעורר

1. לפתוח את `אורינטציה_אפליקציה.html` ולשחק עם שתי הכניסות (מתמחה + מנהל) — לראות שהכל מתנהג כמצופה.
2. לסמן לי מה לכוונן (טקסטים, תחנות, כמויות בצ'קליסט, טיפים, סדר השבועות).
3. אם מתאים — אקים יחד איתך את Supabase (מסלול C) ואת ההעלאה ל-Netlify, ונהיה חיים ברשת באותו היום.
