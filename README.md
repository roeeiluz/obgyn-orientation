# אפליקציית אורינטציה — התמחות נשים ומיילדות (OB/GYN)

מערכת רשת לניהול ומעקב של חודש הקליטה (אורינטציה) למתמחה חדש/ה בחטיבת נשים ומיילדות. כלי אחד, שקוף ומבוסס אמון, שמלווה את המתמחה לאורך החודש — מהלו"ז היומי, דרך יעדי ההכשרה והלמידה העיונית, ועד שער ההסמכה לתורן רביעי ותיק הביצוע בסיום.

מנהל התוכנית: ד"ר אילוז רועי.

---

## הפעלה

האפליקציה היא **קובץ HTML יחיד עצמאי** — `index.html`. אין צורך בהתקנה או בשרת:

- פתיחה מקומית: לחיצה כפולה על `index.html` בדפדפן.
- ברשת: ראו "פריסה" למטה.

בכניסה בוחרים תפקיד — **מתמחה** או **מנהל תוכנית**. המערכת שקופה: כל אחד רואה מה כל אחד עושה (אין מידע אישי רגיש; זו תוכנית לימוד).

---

## יכולות עיקריות

- **תצוגת לוח חודשי** — הלוח המשותף של המחזור, צבע לכל מתמחה, הדגשת הימים של המתמחה הנוכחי.
- **ניהול תוכנית (מנהל)** — הגדרות מחזור, מתמחים (עד 3), **בונה תוכנית** (צ'קליסט שמשבץ פעילויות לתאריכים בלוח), עריכת לו"ז בטבלה, ושיבוץ מתמחים ליום בלחיצה.
- **יעדי הכשרה** — שלוש שכבות מתקפלות: מיומנויות קליניות (סטטוס), התנסות מודדת (מונים), ושליטה בחומר העיוני (נגזר מהלמידה).
- **למידה עיונית** — ספריית מקורות לפי שני צירים נפרדים: **סוג** (פרוטוקול / נוהל-פק"ל / נייר עמדה / ספר לימוד / ערכת הדרכה) ו**רמת שליטה** (מלאה / ידיעה / לקריאה). פתיחת קבצים בלחיצה, סינון, סימון "נשלט", והערות.
- **הסמכה לתורן רביעי** — שער סיום שמתעדכן אוטומטית מיעדי ההכשרה + מבחן וחתימות.
- **התקדמות, הערות יומיות ומשוב** — המתמחה כותב הערות סוף-יום שמגיעות לריכוז ההערות בבית של המנהל (סימון "נקרא" → נשמר בתיק המתמחה).
- **ארכיון** — סגירת תוכנית בסיום שומרת תיק ביצוע מלא ומסירה את המתמחה מכל המסכים.
- **ייצוא** — לוח ליומן (.ics), תוכנית ל-CSV, ותיק ביצוע מתמחה.
- מותאם לנייד ולמחשב, עברית RTL.

## נתונים ושמירה

- ברירת מחדל: הנתונים נשמרים מקומית בדפדפן (localStorage).
- אופציונלי: סנכרון ענן חי בין מכשירים דרך **Supabase** — ראו `README_פריסה.md` ו-`supabase_schema.sql`. מילוי שני ערכים ב-`CONFIG` בראש הסקריפט מפעיל את הסנכרון; השארה ריקה = מצב מקומי.

---

## פריסה (GitHub Pages)

1. העלאת `index.html` ל-repository.
2. Settings → Pages → Branch: `main` → שמירה. תוך דקה מתקבל קישור ציבורי.
3. למצב חי רב-מכשירים: חיבור Supabase לפי `README_פריסה.md`.

הערה: קבצי הפרוטוקולים, ניירות העמדה והערכות גדולים (~30MB) ואינם מוטמעים ב-HTML. כדי שהקישורים יעבדו באתר החי יש להעלות גם את תיקיות המקורות, או לחבר אותן ל-Supabase Storage.

## קבצים ב-repository

- `index.html` — האפליקציה (קובץ יחיד).
