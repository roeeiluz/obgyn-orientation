# סנכרון ענן חינמי דרך Google Apps Script

חלופה ל-Supabase, ללא הרשמה לשירות נוסף — הכל יושב בחשבון Google שלך. מתמחה
רושם הערה בטלפון → היא מופיעה אצלך במסך, וכולם רואים אותו לוח מעודכן. הסנכרון
נעשה בעדכון (polling) כל כ-5 שניות (אין כאן websockets כמו ב-Supabase, אבל
לצרכים של 1–3 משתמשים זה חלק לחלוטין).

## למה זה עובד גם בלי שרת
ה-Apps Script הוא "Web App" קטן שמאזין בכתובת `/exec`:
- `GET` מחזיר את מצב האפליקציה (אובייקט DB אחד) כ-JSON.
- `POST` עם `{"data": ...}` שומר מצב חדש.

המצב נשמר ב-**Script Properties** (מחולק למקטעים כדי לעקוף את מגבלת ה-9KB). אין
צורך בגיליון, בקובץ או במסד נתונים.

---

## הקמה (פעם אחת, ~5 דקות)

### שלב 1 — יצירת הסקריפט
1. היכנס/י ל-[script.google.com](https://script.google.com) → **New project**.
2. מחק/י את התוכן ב-`Code.gs` והדבק/י את כל תוכן הקובץ `apps_script/Code.gs` מכאן.
3. שמור/י (אייקון הדיסקט).

### שלב 2 — פריסה כ-Web App
1. **Deploy → New deployment**.
2. ליד "Select type" לחץ/י על גלגל השיניים → **Web app**.
3. הגדרות:
   - **Execute as:** `Me`
   - **Who has access:** `Anyone`  ← **חובה.** אם זה מוגדר "Only myself"
     הדפדפן יקבל שגיאת **403** והסנכרון לא יעבוד.
4. **Deploy** → אשר/י את ההרשאות (בחר/י את חשבון Google שלך → Advanced →
   Go to project → Allow).
5. העתק/י את ה-**Web app URL** שמסתיים ב-`/exec`.

### שלב 3 — חיבור האפליקציה
1. פתח/י את `index.html` בעורך טקסט.
2. בראש קטע ה-`<script>` יש בלוק `CONFIG`. הדבק/י את הכתובת:
   ```js
   var CONFIG={
     GAS_URL:'https://script.google.com/macros/s/.../exec',
     SUPABASE_URL:'', SUPABASE_KEY:''
   };
   ```
3. שמור/י. מרגע זה האפליקציה טוענת ושומרת בענן ומתרעננת חי בין מכשירים.
   השארת `GAS_URL` ריק = חזרה אוטומטית למצב מקומי.

> כבר מוגדרת כתובת ברירת מחדל ב-`index.html`. אם פרסת/ה מחדש (כל "New
> deployment" נותן כתובת חדשה) — עדכן/י את `GAS_URL` בהתאם.

## עדכון הקוד אחרי שינוי
Apps Script נועל גרסת פריסה. אחרי עריכת `Code.gs`:
**Deploy → Manage deployments → עיפרון (Edit) → Version: New version → Deploy.**
כך הכתובת נשארת זהה. (לעומת זאת "New deployment" יוצר כתובת חדשה.)

## אבחון תקלות
- **403 / דף התחברות:** ה-Web app לא מוגדר "Anyone". חזור/חזרי לשלב 2.3.
- **הנתונים לא מסונכרנים:** פתח/י קונסול בדפדפן (F12) וחפש/י `gas push` /
  `gas pull`. שגיאת CORS לרוב פירושה כתובת שגויה או פריסה ישנה.
- **איפוס הענן:** ב-Apps Script הרץ/הריצי פונקציה שמוחקת את ה-Script
  Properties, או פשוט שמור/י מצב חדש מהאפליקציה (כתיבה דורסת את הקודם).

## אבטחה ופרטיות
כמו ב-Supabase: אין כאן מידע רפואי של מטופלים. הכתובת ציבורית — כל מי שיש לו
אותה יכול לקרוא ולכתוב. זה תואם למודל השקוף ומבוסס-האמון של התוכנית. לבקרה
אמיתית (התחברות אישית, מנהל בלבד עורך) אפשר להוסיף בעתיד שכבת הרשאות.
